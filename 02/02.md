---

# ⏱️ Route Params (Parâmetros de Rota)

- 🔍 Identificador: `:id`
- 📥 Acesso: `req.params`

> São obrigatórios e servem para identificar um recurso. Fazem parte da rota.
>
> Exemplo: `/perfil/:codigo_user`
> 

---

# ⏱️ Query Params (Query Strings)

- 🔍 Identificador: `?chave=valor`
- 📥 Acesso: `req.query`

> São opcionais e servem para filtrar ou buscar dados.
>
> Exemplo: `youtube.com/watch?v=cod_video`
> 

---

## 💡 Exercício de Observação (Youtube)

1. Entre no [Youtube](https://www.youtube.com).
2. Clique em qualquer vídeo e **observe a URL** lá no topo do navegador.
3. Você verá algo como `.../watch?v=XXXXXXXX`.
4. Agora, clique em um **vídeo diferente**.
5. Note que a URL mudou apenas o valor após o `v=`.

> **Conclusão**: O Youtube usa Query Params para dizer ao servidor qual vídeo deve ser carregado sem precisar mudar a página (`/watch`)!

# 🧪 Prática: Tudo em um arquivo

> Exemplo de como o código se parece no `app.js`:
> 

```javascript
const express = require("express");
const app = express();

app.use(express.json());

// 1. Rota com Params (Obrigatório)
app.get("/perfil/:id", (req, res) => {
  const { id } = req.params;
  res.send(`Perfil do usuário: ${id}`);
});

// 2. Rota com Query (Opcional)
app.get("/videos", (req, res) => {
  const { v } = req.query; // Ex: /videos?v=123
  res.json({ msg: `Assistindo o vídeo ID: ${v}` });
});

app.listen(3000, () => console.log("Servidor ON"));
```
