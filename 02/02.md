# ⏱️Server + App

## 📁 Criar estrutura

- 📁Pasta do projeto:
    - `src`
- 📄 Criar arquivos principais:
    - `app.js`
    - `server.js`

---

## 🧪 server.js

> Exemplo:
> 

```
const app = require("./app");

app.listen(3000, () => {
console.log("Servidor rodando");
});
```

---

## 🧪 app.js

> Exemplo:
> 

```
const express=require("express");

const app=express();

app.use(express.json());

module.exports = app;
```

# ⏱️ Routes (Rotas)

- 📁Pasta:
    - `routes`
- 📄 Arquivo:
    - `userRoutes.js`

> Responsáveis por definir os pontos de entrada (endpoints) da nossa API.
>
> Elas determinam qual URL e qual método HTTP (GET, POST, etc) o usuário deve acessar para realizar uma operação, encaminhando a requisição para o Controller adequado.
> 

---

# ⏱️ Controllers

- 📁Pasta:
    - `controllers`
- 📄 Arquivo:
    - `userController.js`

> Responsáveis por receber as requisições HTTP interpretá-las e chamar as funções adequadas dos serviços correspondentes. 

Eles lidam com a entrada e saída da aplicação, mapeando os endpoints da API para as funções apropriadas. 

Em um contexto RESTful, um controller seria responsável por cada rota da API.
>
