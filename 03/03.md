# ⏱️ Introdução Recap

## Recap

Na aula passada, vimos como capturar dados dinâmicos usando `req.params` e `req.query`. No entanto, colocamos tudo dentro de um único arquivo: o `app.js`. 

---

## 📌 Relembrar rapidamente

- `req.params`: Usado para IDs e dados obrigatórios na URL.
- `req.query`: Usado para filtros e buscas opcionais.
- **O Problema:** Conforme nossa API cresce, o `app.js` vira uma "tripa" gigante de código, impossível de dar manutenção.

---

## Modelagem (Monolito Modular)

> Dividir o código para que várias pessoas possam trabalhar juntas sem se "atropelarem" e para que a manutenção seja simples.
---

# 🚀 Passo a Passo

Vamos transformar nosso projeto em algo profissional. Siga os passos abaixo:

### 📁 Passo 1: Criando as Pastas
No seu VS Code, crie duas novas pastas na raiz do projeto:
1.  **`controllers`**: Onde ficará a inteligência (lógica) da sua API.
2.  **`routes`**: Onde ficarão os caminhos (URLs).

---

### 📄 Passo 2: O Controller (`controllers/user.controller.js`)
O Controller é o "cérebro". Ele decide o que fazer com os dados. 
Crie o arquivo `user.controller.js` dentro da pasta `controllers`:

```javascript
// A lógica que antes ficava no app.get agora vive aqui
const boasVindas = (req, res) => {
  const { nome } = req.params;
  res.json({ 
    msg: `Boas vindas ${nome}!`,
    status: "Sucesso"
  });
};

module.exports = {
  boasVindas
};
```

---

### 📄 Passo 3: A Rota (`routes/user.routes.js`)
A Rota é o "porteiro". Ela apenas diz: "Se alguém vier neste caminho, chame aquela função do Controller". Crie o arquivo `user.routes.js` dentro da pasta `routes`:

```javascript
const express = require("express");
const router = express.Router();
const userController = require("../controllers/user.controller");

// Definimos o caminho e associamos ao Controller
router.get("/:nome", userController.boasVindas);

module.exports = router;
```

---

### 📄 Passo 4: Conectando Tudo (`app.js`)
Agora, nosso `app.js` ficou limpo e organizado. Ele serve apenas para configurar o Express e "importar" nossas rotas.

```javascript
const express = require("express");
const userRoutes = require("./routes/user.routes");

const app = express();
app.use(express.json());

// Usamos as rotas de usuário
app.use("/usuarios", userRoutes);

module.exports = app;
```

---

### 📄 Passo 5: O Servidor (`server.js`)
Para ser 100% profissional, separamos a configuração do app da inicialização do servidor. Crie o arquivo `server.js`:

```javascript
const app = require("./app");

const PORT = 3000;

app.listen(PORT, () => {
  console.log(`🚀 Servidor voando na porta ${PORT}`);
});
```

---

# 🧪 Prática Final

Para testar se tudo funcionou:
1. Abra o terminal e rode: `node server.js`
2. Acesse no seu navegador: `http://localhost:3000/usuarios/Matheus`

> **O que aconteceu?**
> O `server.js` ligou o motor -> O `app.js` direcionou para `/usuarios` -> O `user.routes.js` viu o nome -> O `user.controller.js` respondeu com o JSON.
